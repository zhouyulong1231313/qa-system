## 方案1️⃣：Windows Server + IIS 部署

### 适用环境
- Windows Server 2012/2016/2019/2022
- 已安装 IIS (Internet Information Services)

### 部署步骤

#### 第一步：确认 IIS 已安装

1. 按 `Win + R`，输入 `inetmgr`，按回车
2. 如果能打开 IIS 管理器，说明已安装
3. 如果没有，需要先安装 IIS：
   - 打开"服务器管理器"
   - 添加角色和功能
   - 勾选"Web 服务器 (IIS)"
   - 完成安装

#### 第二步：创建网站目录

1. 在服务器上创建文件夹（推荐位置）：
   ```
   C:\inetpub\wwwroot\qa-system\
   ```

2. 将本部署包中的所有文件复制到该文件夹

#### 第三步：在 IIS 中创建网站

1. 打开 IIS 管理器
2. 右键点击"网站" → "添加网站"
3. 填写信息：
   ```
   网站名称: QA-System
   物理路径: C:\inetpub\wwwroot\qa-system
   绑定类型: http
   IP地址: (选择服务器内网IP，如 192.168.1.100)
   端口: 80 (或其他未占用端口，如 8080)
   主机名: (留空或填写 qa.公司域名)
   ```
4. 点击"确定"

#### 第四步：配置 MIME 类型（重要）

1. 在 IIS 管理器中选择新建的网站
2. 双击"MIME 类型"
3. 确认包含以下类型（通常已存在）：
   ```
   .html  →  text/html
   .json  →  application/json
   .js    →  application/javascript
   ```

#### 第五步：测试访问

1. 在服务器上打开浏览器
2. 访问：`http://localhost/qa.html`
3. 如果正常显示，部署成功！

#### 第六步：员工访问

员工在浏览器输入：
```
http://服务器内网IP/qa.html
例如：http://192.168.1.100/qa.html
```

## 方案2️⃣：Linux + Nginx 部署

### 适用环境
- Ubuntu / CentOS / Debian 等 Linux 系统
- 已安装或将安装 Nginx

### 部署步骤

#### 第一步：安装 Nginx（如未安装）

**Ubuntu/Debian:**
```bash
sudo apt update
sudo apt install nginx -y
```

**CentOS/RHEL:**
```bash
sudo yum install nginx -y
```

#### 第二步：创建网站目录

```bash
sudo mkdir -p /var/www/qa-system
```

#### 第三步：上传文件

将部署包中的所有文件上传到服务器：

**方法A：使用 SCP（从你的电脑执行）**
```bash
scp -r deploy-package/* 用户名@服务器IP:/var/www/qa-system/
```

**方法B：使用 FTP 工具**
- 使用 FileZilla 或 WinSCP
- 连接到服务器
- 上传所有文件到 /var/www/qa-system/

#### 第四步：设置文件权限

```bash
sudo chown -R www-data:www-data /var/www/qa-system
sudo chmod -R 755 /var/www/qa-system
```

#### 第五步：配置 Nginx

创建配置文件：
```bash
sudo nano /etc/nginx/sites-available/qa-system
```

粘贴以下内容：
```nginx
server {
    listen 80;
    server_name _;  # 或改为你的内网域名，如 qa.company.local

    root /var/www/qa-system;
    index qa.html index.html;

    location / {
        try_files $uri $uri/ =404;
    }

    # 确保 JSON 文件返回正确的 Content-Type
    location ~* \.json$ {
        add_header Content-Type application/json;
    }

    # 缓存静态文件
    location ~* \.(html|css|js|json)$ {
        expires 1h;
        add_header Cache-Control "public, must-revalidate";
    }
}
```

#### 第六步：启用网站

```bash
# 创建符号链接
sudo ln -s /etc/nginx/sites-available/qa-system /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx

# 设置开机自启
sudo systemctl enable nginx
```

#### 第七步：配置防火墙

```bash
# Ubuntu
sudo ufw allow 80/tcp
sudo ufw reload

# CentOS
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --reload
```

#### 第八步：测试访问

在服务器上测试：
```bash
curl http://localhost/qa.html
```

如果返回 HTML 内容，部署成功！

#### 第九步：员工访问

员工在浏览器输入：
```
http://服务器内网IP/qa.html
例如：http://192.168.1.100/qa.html
```

## 方案3️⃣：Linux + Apache 部署

### 适用环境
- Linux 系统
- 已安装或将安装 Apache

### 部署步骤

#### 第一步：安装 Apache（如未安装）

**Ubuntu/Debian:**
```bash
sudo apt update
sudo apt install apache2 -y
```

**CentOS/RHEL:**
```bash
sudo yum install httpd -y
```

#### 第二步：创建网站目录

```bash
sudo mkdir -p /var/www/qa-system
```

#### 第三步：上传文件

将所有文件上传到 `/var/www/qa-system/`

#### 第四步：设置权限

```bash
sudo chown -R www-data:www-data /var/www/qa-system  # Ubuntu
# 或
sudo chown -R apache:apache /var/www/qa-system      # CentOS

sudo chmod -R 755 /var/www/qa-system
```

#### 第五步：配置虚拟主机

**Ubuntu/Debian:**
```bash
sudo nano /etc/apache2/sites-available/qa-system.conf
```

**CentOS:**
```bash
sudo nano /etc/httpd/conf.d/qa-system.conf
```

粘贴以下内容：
```apache
<VirtualHost *:80>
    DocumentRoot /var/www/qa-system

    <Directory /var/www/qa-system>
        Options Indexes FollowSymLinks
        AllowOverride None
        Require all granted

        DirectoryIndex qa.html index.html
    </Directory>

    # 确保 JSON 正确 MIME 类型
    AddType application/json .json

    ErrorLog ${APACHE_LOG_DIR}/qa-system-error.log
    CustomLog ${APACHE_LOG_DIR}/qa-system-access.log combined
</VirtualHost>
```

#### 第六步：启用网站

**Ubuntu/Debian:**
```bash
sudo a2ensite qa-system.conf
sudo systemctl restart apache2
sudo systemctl enable apache2
```

**CentOS:**
```bash
sudo systemctl restart httpd
sudo systemctl enable httpd
```

#### 第七步：配置防火墙

```bash
# Ubuntu
sudo ufw allow 'Apache'

# CentOS
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --reload
```

#### 第八步：员工访问

```
http://服务器内网IP/qa.html
```

## 🔧 部署后配置

### 1. 设置默认页面（可选）

如果希望访问根路径自动打开 qa.html：

**Nginx:** 在配置中已包含 `index qa.html`

**IIS:**
1. 选择网站 → 双击"默认文档"
2. 添加 `qa.html` 并移到最上面

**Apache:** 在配置中已包含 `DirectoryIndex qa.html`

### 2. 绑定内网域名（可选）

如果公司有内网 DNS，可以配置域名：
```
qa.公司域名  →  服务器IP
例如：qa.company.local → 192.168.1.100
```

员工就可以访问：`http://qa.company.local`

### 3. 配置访问控制（可选）

**限制只有内网可访问：**

**Nginx:**
```nginx
location / {
    allow 192.168.0.0/16;  # 允许内网
    deny all;              # 拒绝其他
    try_files $uri $uri/ =404;
}
```

**Apache:**
```apache
<Directory /var/www/qa-system>
    Require ip 192.168.0.0/16
</Directory>
```

## 📝 更新问答数据

### 方法1：手动更新

1. 在管理后台导出新的数据文件
2. 替换服务器上的 `qa-data.json`
3. 通知员工刷新浏览器

### 方法2：自动同步（高级）

如果服务器能访问 GitHub，可以设置定时同步：

**创建更新脚本 `update.sh`:**
```bash
#!/bin/bash
cd /var/www/qa-system
git pull origin main
sudo systemctl reload nginx  # 或 apache2/httpd
```

**设置定时任务（每小时更新一次）:**
```bash
crontab -e
# 添加：
0 * * * * /path/to/update.sh
```

## 🔍 故障排查

### 问题1：无法访问网站

**检查服务是否运行：**
```bash
# Nginx
sudo systemctl status nginx

# Apache (Ubuntu)
sudo systemctl status apache2

# Apache (CentOS)
sudo systemctl status httpd
```

**检查防火墙：**
```bash
# Ubuntu
sudo ufw status

# CentOS
sudo firewall-cmd --list-all
```

### 问题2：404 错误

检查文件路径和权限：
```bash
ls -la /var/www/qa-system/
```

### 问题3：JSON 数据加载失败

**查看浏览器控制台：**
- 按 F12 打开开发者工具
- 查看 Console 和 Network 标签

**检查 MIME 类型：**
```bash
# 测试 JSON 文件
curl -I http://localhost/qa-data.json
```

应该返回：`Content-Type: application/json`

### 问题4：权限问题

```bash
# Linux - 重新设置权限
sudo chown -R www-data:www-data /var/www/qa-system
sudo chmod -R 755 /var/www/qa-system
```
