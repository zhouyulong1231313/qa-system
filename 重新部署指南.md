# QA系统 Vercel 重新部署指南

## 准备工作
已为你准备好包含10条新数据的部署包，位于：
`C:\Users\wy_zhouyulong\qa-system\deploy-package`

---

## 第一步：创建 GitHub 账号（5分钟）

1. 访问：https://github.com/signup
2. 输入邮箱地址（建议使用常用邮箱）
3. 创建密码（至少15个字符，或8个字符+数字和字母）
4. 输入用户名（可以是任意英文名）
5. 完成验证后点击"Create account"
6. 检查邮箱，点击验证链接
7. 完成！记住你的用户名和密码

---

## 第二步：创建新的 GitHub 仓库（2分钟）

1. 登录 GitHub 后，点击右上角 "+" → "New repository"
2. 填写仓库信息：
   - Repository name: `qa-system`（或其他你喜欢的名字）
   - Description: `智能问答系统`
   - 选择 **Public**（公开）
   - **不要勾选** "Add a README file"
3. 点击 "Create repository"
4. 看到新页面后，**复制仓库地址**（格式：https://github.com/你的用户名/qa-system.git）

---

## 第三步：上传代码到 GitHub（3分钟）

1. 打开命令行（Win+R 输入 cmd）

2. 进入部署包目录：
```bash
cd C:\Users\wy_zhouyulong\qa-system\deploy-package
```

3. 初始化 Git（如果还没有）：
```bash
git init
git add .
git commit -m "初始化QA系统"
```

4. 连接到你的 GitHub 仓库（替换下面的地址为你刚才复制的地址）：
```bash
git remote add origin https://github.com/你的用户名/qa-system.git
git branch -M main
git push -u origin main
```

5. 输入你的 GitHub 用户名和密码（或 Personal Access Token）

6. 看到 "successfully" 表示上传成功！

---

## 第四步：创建 Vercel 账号并部署（3分钟）

1. 访问：https://vercel.com/signup

2. 点击 "Continue with GitHub" 用 GitHub 账号登录

3. 授权 Vercel 访问你的 GitHub 账号

4. 登录成功后，点击 "Add New..." → "Project"

5. 在列表中找到 `qa-system` 仓库，点击 "Import"

6. 配置项目：
   - Framework Preset: 选择 **Other**
   - Root Directory: `.` （保持默认）
   - Build Command: 留空
   - Output Directory: `.` （保持默认）
   - Install Command: 留空

7. 点击 "Deploy" 开始部署

8. 等待1-2分钟，看到 "Congratulations!" 表示部署成功

9. 复制你的新网站地址（格式：https://xxx.vercel.app）

10. 访问：
    - 用户页面：https://你的域名.vercel.app/qa.html
    - 管理页面：https://你的域名.vercel.app/admin.html

```bash
cd C:\Users\wy_zhouyulong\qa-system\deploy-package

```
3. Vercel 会自动检测更新并重新部署（1-2分钟）

## 遇到问题？

- **GitHub 推送失败**：检查用户名密码是否正确，或使用 Personal Access Token
- **Vercel 部署失败**：检查文件是否完整，或联系支持
- **网站无法访问**：清除浏览器缓存后重试
