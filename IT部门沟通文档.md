# IT部门沟通文档

## 项目概述

**项目名称：** 智能问答系统
**项目类型：** 纯静态网站
**技术栈：** HTML + CSS + JavaScript
**数据存储：** JSON文件 + 浏览器LocalStorage

---

## 部署需求

### 资源需求
- **存储空间：** 约 50 MB
- **端口需求：** HTTP 80 端口（或其他可用端口）
- **访问方式：** 仅需内网访问
- **服务器要求：** 支持静态文件托管即可（IIS/Nginx/Apache 均可）

### 文件清单
```
qa.html              # 用户问答主页面
admin.html           # 管理后台
index.html           # 备用用户界面
index-chat.html      # 聊天式界面
qa-data.json         # 问答数据文件
```

---

## 安全性说明

✅ **无安全风险：**
- 纯静态HTML文件，无服务器端代码
- 无数据库连接
- 无文件上传功能
- 无敏感信息存储在服务器
- 用户交互数据仅存储在用户本地浏览器

✅ **内网隔离：**
- 仅需内网访问
- 无需外网暴露

---

## 推荐部署方式

### Windows Server (IIS)
1. 创建目录：`C:\inetpub\wwwroot\qa-system\`
2. 复制所有文件到该目录
3. 在IIS中创建新网站，指向该目录
4. 绑定内网IP和端口（建议80或8080）

**访问地址：** `http://服务器IP/qa.html`

### Linux (Nginx/Apache)
1. 创建目录：`/var/www/qa-system/`
2. 上传所有文件
3. 配置虚拟主机（详见完整部署指南）
4. 重启Web服务

**访问地址：** `http://服务器IP/qa.html`

---

## 配置要点

### MIME类型配置（重要）
确保服务器正确识别JSON文件：
- `.json` → `application/json`
- `.html` → `text/html`
- `.js` → `application/javascript`

### 默认文档设置（可选）
将 `qa.html` 设置为默认文档，用户访问根路径时自动打开问答页面。
